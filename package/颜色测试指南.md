# Mind 颜色分析测试指南

本文档用示例 .mind 文件展示各词汇类型的颜色。
按 **AI修正颜色** 按钮后，AI 会分析每个词汇并分配颜色。

---

## 1. 测试文件

创建 `test_color.mind`，写入以下内容：

```mind
# 链表反转 — 中文注释
Define function reverse_list
input: head node of linked list
output: reversed head node

if head is null or head.next is null
  return head

# 双指针法遍历
set prev = null
set curr = head

while curr is not null
  save next_node = curr.next
  curr.next = prev
  prev = curr
  curr = next_node

return prev

# 测试代码
count = 0
items = [1, 2, 3, 4, 5]
for each item in items
  count = count + item
print "total:"
print count

if count > 15
  print "large"
else
  print "small"

# @d 需要修改: 增加异常处理
```

## 3. AI 装饰标记

按 **AI修正颜色** 后，**被 AI 处理过的每一行**左侧会出现：

> ░ 浅紫色半透明背景 + 细边框
> 表示：这行已被 AI 分析并修正颜色

- 注释行、空行没有装饰
- 再次按 **AI修正颜色** → 旧装饰先清除，新装饰再添加

## 4. 按行预览

```mind
  ░ # 链表反转 — 中文注释                 ← 灰绿 (comment)
  ░ Define function reverse_list          ← 蓝 (declaration) + 亮蓝 (function)
  ░ input: head node of linked list       ← 青色 (nouns)
  ░ output: reversed head node            ← 青色 (nouns)
  (空行，无装饰)
  ░ if head is null or head.next is null  ← 蓝 (keyword) + 青 (variable)
  ░   return head                         ← 蓝 (keyword) + 青 (variable)
  (空行)
  ░ # 双指针法遍历                         ← 灰绿 (comment)
  ░ set prev = null                       ← 蓝 (verb) + 青 (variable) + 浅绿 (const)
  ░ set curr = head                       ← 蓝 (verb) + 青 (variable)
  (空行)
  ░ while curr is not null                ← 蓝 (keyword) + 青 (variable)
  ░   save next_node = curr.next          ← 蓝 (verb) + 青 (variable) + 白 (operator)
  ░   curr.next = prev                    ← 青 (variable) + 白 (operator) + 青 (variable)
  ░   prev = curr                         ← 青 (variable) + 白 (operator) + 青 (variable)
  ░   curr = next_node                    ← 青 (variable) + 白 (operator) + 青 (variable)
  (空行)
  ░ return prev                           ← 蓝 (keyword) + 青 (variable)
  (空行)
  ░ # @d 需要修改: 增加异常处理            ← 灰绿 (comment，带 @d 也是注释)
```

## 5. 清除 AI 标记

按 **清除AI标记**：
- 所有 AI 分配的颜色被移除
- 恢复为 TextMate 语法着色（基础色方案）
- 浅紫色装饰消失
- 可再次按 **AI修正颜色** 重新分析

## 6. Token 用量

每次分析后：
- 状态栏更新 Token 累计消耗
- 点 **状态栏 Mind** → 打开 Token 面板
  - 总 Token 数
  - 缓存命中率（百分比 + 进度条）
  - 估算费用（USD + CNY）
- Token 数据存在 VS Code 全局存储，重启不丢失
