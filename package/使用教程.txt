Mind 自然语言语义标注系统 — 安装使用说明
==============================================

📁 安装包内容
─────────────
  Mind-语言系统-安装包/
  ├── start.bat             ← ★ 双击这个开始
  ├── 使用教程.txt           ← 本文件
  ├── 示例.mind             ← 功能演示文件
  ├── mind-bridge/          ← 桥接服务（链接到实际目录）
  └── mind-extension/       ← VS Code 插件（链接到实际目录）


🚀 快速开始（三步）
─────────────────

  第一步：启动桥接服务
    双击「start.bat」→ 选 [1] 一键安装 + 启动
    或手动: 打开命令行 → cd mind-bridge → npm start

  第二步：安装 VS Code 插件
    在启动器菜单选 [4] 打包 VS Code 插件
    → 生成 .vsix 文件 → VS Code 中安装
    或: 用 VS Code 打开 mind-extension 文件夹 → 按 F5

  第三步：开始使用
    创建任意 .mind 文件，开始写你的需求！


📝 .mind 文件语法
─────────────────

  # 注释              ← Python 风格，灰绿色显示
  # @d 实现XX         ← 特殊注释：触发代码生成（Python + C）
  if/else/while/for   ← 控制流关键字（深蓝色）
  func()              ← 函数调用（亮蓝色）
  count/result/node   ← 变量/实体（浅蓝色）
  int/float/void      ← 类型关键字（青色）
  30/0.5/TRUE/NULL    ← 数字/常量（浅绿色）
  "text"/'c'          ← 字符串（橙色）
  → / ->              ← 指针/箭头（白色）


🎯 三文件体系
─────────────
  当你创建 project.mind 并写入 # @d 需求后，
  插件自动在同目录创建：

  project.mind  ← 你写的需求规格
  project.py   ← 自动生成的 Python 实现（含详细注释）
  project.c    ← 自动生成的 C 实现（含详细注释）

  三个文件之间有超链接连接：
  • 在 .mind 中点击 # @d 注释 → 跳转到 .py/.c 的函数
  • 在 .py/.c 中点击函数注释 → 跳回 .mind 的需求


🔄 工作流程
───────────

  1. 打开 .mind 文件
  2. 写一行需求描述
  3. 按 Enter → 插件自动分析整份文件
     ✓ 词汇上色（18种类型）
     ✓ 缩进关系线（├─ └─ │）
     ✓ 实体关联（悬停高亮）
     ✓ 逻辑诊断（红/绿波浪线）
  4. 写 # @d 注释 → 插件自动生成实现代码
     ✓ project.py 更新
     ✓ project.c 更新
     ✓ 超链接创建
  5. 鼠标悬停 → 灰色高亮同义词 + 弹窗实体信息
     ✓ 零额外 API 请求（数据已缓存）
  6. 状态栏显示累计 Token 消耗


⚙️ 设置
───────
  VS Code → 文件 → 首选项 → 设置 → 扩展 → Mind Language

  可自定义:
  • 每种词汇的颜色（颜色选择器）
  • 缩进线颜色（按层级）
  • 缩进线/悬停/诊断 开关
  • 桥接服务端口


💡 常见问题
───────────

  Q: 插件没反应？
  A: 检查 VS Code 右下角语言模式是否为 "Mind"
     检查桥接服务是否在运行（curl localhost:3456/health）

  Q: 状态栏显示 "(离线)"？
  A: 桥接服务未启动，运行「start.bat」选 [3]

  Q: 没有颜色高亮？
  A: 文件后缀必须为 .mind。TextMate 语法提供基本高亮
     无需桥接服务也能工作

  Q: # @d 没生成代码？
  A: 确保写完 # @d 所在行后按了 Enter
     检查桥接服务日志是否有错误

  Q: 如何查看 Token 消耗？
  A: 状态栏显示 "Mind 🗄XXXXT"
     悬停状态栏查看详细
